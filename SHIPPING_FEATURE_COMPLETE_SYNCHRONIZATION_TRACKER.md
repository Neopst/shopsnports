### 3️⃣ FIRESTORE - DATABASE SCHEMA

**Collection:** `shippingRequests`  
**Files:**
- `FIRESTORE_COLLECTIONS_SCHEMA_2026.md` ✅
- `INITIALIZE_SHIPPING_COLLECTION.md` ✅

**Status:** ✅ COMPLETE

**Document Structure (Verified):**
```javascript
{
  // Identifiers
  id: string,                      // Auto-generated by Firestore
  requesterId: string,             // Firebase Auth UID
  affiliateId: string | null,      // Referral partner (optional)
  
  // Status & Tracking
  status: 'pending' | 'approved' | 'in_transit' | 'delivered' | 'cancelled',
  trackingNumber: string | null,   // Generated after approval (SHP-YYYYMMDD-XXXXX)
  
  // 1. Freight Type
  freightType: 'door_to_door' | 'airport_to_airport',
  
  // 2. Shipment Details
  itemDescription: string,         // What's being shipped
  hsCode: string | null,           // Harmonized tariff code
  departingLocation: string,       // Origin city
  departureDate: Timestamp,        // Expected departure
  destinationLocation: string,     // Destination city
  arrivalDate: Timestamp,          // Expected arrival
  shipmentWeight: number,          // kg
  shipmentLength: number,          // cm
  shipmentWidth: number,           // cm
  shipmentHeight: number,          // cm
  shipmentPackaging: string,       // Box, crate, pallet, etc.
  
  // 3. Sender Details
  senderName: string,
  senderAddress: string,
  senderPhone: string,
  senderEmail: string,
  
  // 4. Receiver Details
  receiverName: string,
  receiverAddress: string,
  receiverPhone: string,
  receiverEmail: string,
  
  // 5. Attachments
  attachments: Array<{
    name: string,
    url: string,
    uploadedAt: Timestamp,
  }>,
  
  // 6. Additional Info
  otherInformation: string | null, // Special requirements
  
  // System Fields (Server Generated)
  assignedAdminId: string | null,  // Admin assigned to request
  estimatedCost: number,           // Quoted price
  actualCost: number,              // Final price (0 until confirmed)
  createdAt: Timestamp,            // Document creation
  updatedAt: Timestamp,            // Last modification
  rejectionReason: string | null,  // If cancelled/rejected
}
```

### 8️⃣ INTEGRATION FLOW - END-TO-END

**Complete User Journey:**

```
1. USER CREATES REQUEST (Mobile App)
   └─> Form submission to Firestore
   └─> Document written to shippingRequests collection
   
2. CLOUD FUNCTION TRIGGERED (onShippingRequestCreated)
   └─> Validate affiliate token
   └─> Create admin notification
   └─> Send FCM to admins
   └─> Send HTML email to customer ✅
   └─> Generate tracking number (SHP-20260302-00001)
   
3. ADMIN RECEIVES NOTIFICATION
   └─> FCM notification on device ✅
   └─> App notification banner
   └─> Updates admin dashboard in real-time ✅
   └─> Admin clicks notification → opens detail screen
   
4. ADMIN REVIEWS & APPROVES
   └─> Clicks "Approve" button
   └─> Status changes: PENDING → APPROVED
   └─> Cloud Function triggered (onShippingRequestUpdated)
   └─> Email sent to customer: "Your request was approved!"
   └─> Tracking number assigned (if not already)
   
5. CUSTOMER SEES UPDATE (Mobile App)
   └─> FCM notification received ✅
   └─> In-app notification displayed ✅ (if implemented)
   └─> User navigates to "Shipping History"
   └─> Request shows status: APPROVED
   └─> Copy tracking number & save
   
6. SHIPMENT IN TRANSIT
   └─> Admin updates status: APPROVED → IN_TRANSIT
   └─> Email sent: "Your shipment is on the way! Tracking: [#]"
   └─> Update visible in mobile app history
   └─> Admin can track milestone
   
7. DELIVERY
   └─> Admin updates status: IN_TRANSIT → DELIVERED
   └─> Final confirmation email sent
   └─> Mobile app shows DELIVERED status
   └─> User can leave feedback/rating
```
