```
{
  'id': STRING,
  'affiliateId': STRING|NULL,
  'userId': STRING|NULL,
  'guestEmail': STRING,
  'freightType': STRING,              ← "door-to-door" or "airport-to-airport"
  'itemDescription': STRING,
  'hsCode': STRING|NULL,
  'departingLocation': STRING,        ← Origin location
  'departureDate': DATETIME,
  'destinationLocation': STRING,      ← Destination
  'arrivalDate': DATETIME,
  'shipmentWeightKg': DOUBLE,
  'shipmentDimensions': STRING,       ← "100x50x30"
  'shipmentPackaging': STRING,
  'senderName': STRING,
  'senderAddress': STRING,
  'senderPhone': STRING,
  'senderEmail': STRING,
  'receiverName': STRING,
  'receiverAddress': STRING,
  'receiverPhone': STRING,
  'receiverEmail': STRING,
  'attachmentUrls': ARRAY<STRING>,
  'otherInformation': STRING|NULL,
  'createdAt': DATETIME,
  'status': STRING,                    ← "pending"
  'adminNotes': STRING|NULL
}
```

### What Firestore Rules REQUIRE (firestore.rules):
```
{
  'requesterId': STRING,              ← ❌ MISSING! (Expected userId or requesterId)
  'clientName': STRING,               ← ❌ Should be senderName
  'clientEmail': STRING,              ← ❌ Should be guestEmail or senderEmail
  'clientPhone': STRING,              ← ❌ Should be senderPhone
  'type': STRING,                     ← ❌ Should be freightType
  'origin': STRING,                   ← ❌ Should be departingLocation
  'destination': STRING,              ← ❌ Should be destinationLocation
  'description': STRING,              ← ❌ Should be itemDescription
  'weight': NUMBER,                   ← ❌ Should be shipmentWeightKg
  'length': NUMBER,                   ← ❌ Missing! (Need separate dimensions)
  'width': NUMBER,                    ← ❌ Missing! (Need separate dimensions)
  'height': NUMBER,                   ← ❌ Missing! (Need separate dimensions)
  'priority': STRING,                 ← ❌ MISSING ENTIRELY
  'status': STRING,                   ← ✅ OK
  'createdAt': TIMESTAMP,             ← ✅ OK
  'trackingNumber': STRING            ← ❌ MISSING! (Must be generated by Cloud Function)
}
```

### Result:
**FIRESTORE REJECTS THE WRITE** because:
- ❌ `requesterId` is missing (required)
- ❌ Field names don't match security rule checks
- ❌ `trackingNumber` not provided upfront
- ❌ Dimensions split wrong (rule expects separate length/width/height)
- ❌ `priority` field missing

## 📋 ISSUE #2: DateTime NOT Properly Serialized

**Location:** `lib/models/shipping_request_simple.dart` - `toFirestore()` method

### Problem:
```dart
// ❌ WRONG - DateTime objects can't be serialized to Firestore directly
'departureDate': departureDate,
'arrivalDate': arrivalDate,
'createdAt': createdAt,
```

### Should be:
```dart
// ✅ CORRECT - Convert to Timestamp
'departureDate': departureDate != null ? Timestamp.fromDate(departureDate) : null,
'arrivalDate': arrivalDate != null ? Timestamp.fromDate(arrivalDate) : null,
'createdAt': Timestamp.fromDate(createdAt),
```

### Impact:
Firestore serialization will fail silently or throw errors, preventing the document from being saved.

### STEP 2: Update Firestore Security Rules
**File:** `firestore.rules`

Replace the shippingRequests validation to accept the new structure:

```plaintext
match /shippingRequests/{requestId} {
  allow read: if 
    isRequester(resource.data.requesterId) ||
    isAdmin() ||
    (resource.data.affiliateId != null && isAffiliate(resource.data.affiliateId));
  
  allow create: if 
    request.resource.data.keys().hasAll([
      'requesterId',
      'clientName',
      'clientEmail',
      'clientPhone',
      'type',
      'origin',
      'destination',
      'description',
      'weight',
      'length',
      'width',
      'height',
      'priority',
      'status',
      'createdAt'
    ]) &&
    request.resource.data.type in ['door-to-door', 'airport-to-airport'] &&
    request.resource.data.clientName.size() > 0 &&
    request.resource.data.clientEmail.matches('.*@.*') &&
    request.resource.data.origin.size() > 0 &&
    request.resource.data.destination.size() > 0 &&
    request.resource.data.description.size() > 0 &&
    request.resource.data.weight > 0 &&
    request.resource.data.length >= 0 &&
    request.resource.data.width >= 0 &&
    request.resource.data.height >= 0 &&
    request.resource.data.status == 'pending' &&
    request.resource.data.priority in ['standard', 'express'];
  
  allow update: if isAdmin();
  allow delete: if isAdmin();
}
```
